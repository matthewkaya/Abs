## Engineering note — regex over substring

First version used `"test.fail(" not in window`. Failed because
the prose comment text `test.fail() upgraded to test()` matches.
Pivoted to a regex tied to the call-site form:

```python
pattern = rf"test\.fail\(\s*[\"']{re.escape(scenario)}"
```

The empty-arg `test.fail()` won't match because there's no
scenario string after the open paren. This is a more robust
pattern for source-grep regression pins — distinguishes prose
from invocation.
