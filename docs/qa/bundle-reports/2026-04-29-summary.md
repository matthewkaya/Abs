## Where the shared 102 kB lives

The `+ First Load JS shared by all` chunk:

```
chunks/255-…js   46 kB    (next-flight runtime + react-dom server bridge)
chunks/4bd1b…js  54.2 kB  (react-dom client + next routing)
other shared chunks (total)   2.07 kB
```

This is the **React 19 + Next 15.5 framework baseline**. Investigated:

- **Phosphor icons** — already loaded via subpath `@phosphor-icons/react/dist/ssr/{Moon,SunHorizon}`. Verified this is per-route, not contributing to the shared chunk (re-build confirmed identical 102 kB after switching).
- **Framer Motion** — only imported by `MetricCard` / `PricingTierCard` / `HeroVisual`, all per-route.
- **R3F + drei + three.js** — `next/dynamic({ ssr: false })` keeps it off the home shared chunk.
- The 2-kB overage *is* the framework. Closing it would require dropping React features (preact-shim) or a custom server-only React build — both are bigger architectural decisions than the launch deadline can absorb.

## Decision

Accept the 2 KB overage; the **per-route first-load JS** sits well within the
150 kB budget for every public page (`/` 109 kB, `/pricing` 110 kB, `/showcase`
145 kB). The Lighthouse perf budget that ships with T-R05 will continue to
gate page-level performance regressions.

## CI gate

`bundlewatch.config.json` ships in `core/landing/`. Hook into CI by adding:

```yaml
- name: Bundle watch
  run: npx bundlewatch --config bundlewatch.config.json
  env:
    BUNDLEWATCH_GITHUB_TOKEN: ${{ secrets.BUNDLEWATCH_GITHUB_TOKEN }}
```

The HTML analyzer report is at `docs/qa/bundle-reports/bundle-report-2026-04-29.html`.
